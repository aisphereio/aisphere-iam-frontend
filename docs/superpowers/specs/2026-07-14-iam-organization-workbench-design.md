# IAM 组织工作台设计

## 目标

面向平台和组织管理员，把现有“组织树、成员详情、组织表单、诊断信息全部纵向堆叠”的页面，重组为一个可定位、可理解、可操作的组织工作台。页面的单一任务是：管理员先在层级中定位对象，再在同一上下文中管理下级组织、成员和组织属性。

## 设计判断

采用“层级导航 + 上下文工作区”的双栏主从结构，不做三栏监控大屏，也不增加后端尚未提供的统计、审计或权限数据。现有 API、React Query hooks、创建弹窗以及用户未提交的相关改动全部保留并复用。

```text
┌──────────────────────────────────────────────────────────────────┐
│ 组织工作台                         [刷新] [新建组织]              │
│ 在一张结构图里管理组织、成员和上下级关系                         │
├──────────────────────┬───────────────────────────────────────────┤
│ 搜索组织或成员       │ Aisphere › 华东区 › 平台研发              │
│ [全部展开][全部收起] │                                           │
│                      │ 平台研发                  [新建下级][编辑] │
│ Aisphere             │ 12 名成员 · 3 个直接下级 · 8 个全部下级  │
│ ├─ 华东区            ├───────────────────────────────────────────┤
│ │  ├─ 平台研发       │ [概览] [成员] [组织设置]                  │
│ │  │  ├─ Alice       │                                           │
│ │  │  └─ Bob         │ 当前标签页的唯一任务区                    │
│ │  └─ 产品设计       │                                           │
└──────────────────────┴───────────────────────────────────────────┘
```

## 视觉系统

- **Orbit Violet `#7C3AED`**：当前路径、选中节点和主要操作，沿用 IAM 品牌色。
- **Directory Ink `#172033`**：标题和关键组织名称，表达目录结构的稳定性。
- **Signal Cyan `#0891B2`**：成员、身份和可进入的下级节点。
- **Topology Amber `#D97706`**：组织文件夹、层级提醒和需要注意的结构信息。
- **Console Cloud `#F7F8FC`**：工作台底色，保持长时间管理操作的低干扰。

字体不新增网络依赖。标题使用现有 Geist Sans 的紧凑粗体，正文使用 Geist Sans 常规字重；路径、ID 和系统字段使用 Geist Mono。层级靠缩进、连接线和图标编码，不靠装饰编号。

## 标志性元素：组织路径脊柱

工作区顶部固定展示从身份源根节点到当前对象的完整路径。每一级都是可点击的上下文切换入口；当前节点使用品牌色强调。它解决三个问题：管理员知道自己在哪、当前操作影响哪个作用域、可以快速返回任意祖先节点。

## 信息架构

### 左侧层级导航

- 搜索同时匹配组织名称、显示名、用户名和邮箱。
- 根节点、组织和成员在同一棵树中呈现，但用图标、字重和标签区分。
- 展开状态和选中状态必须视觉分离；按钮保留可见键盘焦点。
- 空状态直接提供“新建第一个组织”，加载错误提供重试。
- 调试字段诊断移出主流程，只在开发诊断入口中保留。

### 根节点工作区

- 展示身份源名称以及顶级组织、全部组织、成员三个可靠计数。
- 主内容是顶级组织入口，不重复展示整棵树。
- 主要操作是“新建顶级组织”。

### 组织工作区

- 顶部展示组织名称、类型、直接成员、直接下级和全部下级数量。
- **概览**：直接下级组织和当前结构摘要。
- **成员**：成员列表、搜索和添加成员；移除属于危险次级操作。
- **组织设置**：显示名称、组织类型、父级组织；保存与删除分区。
- 新建下级组织继续使用现有创建弹窗，默认父级为当前组织。

### 成员工作区

- 展示身份信息、联系方式、状态和所属组织。
- 提供返回来源组织和进入其他所属组织的入口。
- 不在此处伪造角色或继承权限；权限治理继续由访问控制页负责。

## 交互和反馈

- 创建、编辑、移动、删除均通过明确的弹窗或确认框完成，避免共享表单状态污染当前详情。
- 成功操作保持当前上下文并刷新数据；删除当前组织后回到根节点。
- 请求失败显示原因和重试，不渲染成空目录。
- 搜索无结果时说明搜索范围并提供清除动作。
- 动效只用于工作区内容切换的轻微淡入；遵守 `prefers-reduced-motion`。

## 响应式和可访问性

- `xl` 以上为固定宽度层级栏 + 自适应工作区。
- 平板为较窄双栏；手机改为上下结构，层级导航在工作区之前且不锁定高度。
- 树遵循单选 tree/treeitem 语义，父节点提供 `aria-expanded`，选中节点提供 `aria-selected`。
- 所有图标按钮有可读名称，危险操作不用颜色作为唯一提示。

## 非目标

- 不修改后端协议或 SpiceDB 模型。
- 不新增组织图表、审计时间线、批量导入和拖拽移动。
- 不重做全局侧边栏、用户页或访问控制台。
- 不覆盖当前工作树中已有的创建组织弹窗改动。

## 验收标准

1. 首屏能同时看到组织层级和当前对象工作区，不需要向下寻找主要操作。
2. 根节点、组织、成员三种选择都有明确且不同的工作区。
3. 组织的概览、成员和设置互不混杂；创建、编辑和删除入口清晰。
4. 路径脊柱可返回根节点和祖先组织。
5. 现有创建、更新、移动、删除、添加成员和移除成员能力保持可用。
6. 桌面、平板和手机布局可用，并具备键盘焦点和基础树语义。
